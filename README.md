# 星迹 StarTrail

**基于 Skyfield 天文计算引擎的 Windows 桌面观星工具**

面向零基础观星小白的全中文桌面应用，调用开放 API 获取实时天文与天气数据。新手模式一键观星，专业模式提供完整天文参数。

> **架构说明：** 星迹完全本地运行，仅通过公开 API 查询数据，不部署任何后端服务。所有网络请求均经统一客户端处理，离线时优雅降级为内置天文计算。

- **侧边栏可拖拽调宽**：在左侧栏右边缘（拖拽手柄）按住鼠标左右拖动即可调整侧边栏宽度（180–450px，默认 240px），宽度自动记忆，重启保持。
- **天体资料链接修复**：双击星空图中的天体（恒星/行星/太阳/月亮/深空天体/卫星/飞机）现在会打开对应的权威资料页——恒星/行星走中文维基百科，深空天体走 Aladin Lite 实景星图，卫星走 N2YO，飞机走 Flightradar24 / JetPhotos。修复了此前指向已失效站点（starfyi.com）导致链接打不开的问题。
- **观星适宜度评分显示修正**：移除了与圆形仪表盘重复的「分数 + 评价」文字列，预报页只保留一个清晰的评分仪表盘。
- **未来降水预报修复**：修复了降水卡片始终显示「--」的缓存 bug（预报数据在缓存命中时未被正确保存），现在各时段（2h / 8h / 12h / 24h）降水情况正常显示。
- **全代码库审计与 14 项 bug 修复**：覆盖 `Theme.qcolor` 安全解析（修复多处因 rgba 字符串导致的崩溃）、`astronomy_api` 时区修正、配置损坏容错、DSO / 飞机 / 卫星卡片布局截断、GL 取景框（FOV）叠加等问题。
- **统一 HTTP 客户端 + 响应缓存**：所有 API 调用收口到 `app/api/client.py` 的共享 `Session`（连接池 + 自动重试），并对天气 / NASA / ADS-B 等加 TTL 缓存。非密钥类接口（ip-api、Celestrak、OpenSky）SSL 失败时自动降级 HTTP，密钥类接口（OpenWeatherMap、NASA）永不降级，避免密钥泄露。
- **VPN 安全定位**：自动 IP 定位改为「先探测出口 IP 归属国，再决定是否采用」。当检测到你在中国大陆城市却经由境外 VPN / 代理出口时，**不会覆盖你手动设置的本地坐标**，而是保留手动位置并给出醒目提示。

## 快速开始

### 方式一：下载 EXE（推荐）

1. 前往 [Releases](https://github.com/HazardSun/StarTrail/releases) 下载最新版 `StarTrail_v1.0.2.zip`
2. 解压后双击 `StarTrail.exe`，无需安装 Python 或任何依赖
3. 首次启动会自动下载 JPL 星历表（约 17MB），请保持网络通畅

> **系统要求：** Windows 10/11 64 位，建议 8GB+ 内存，支持 OpenGL 3.3+ 的显卡

### 方式二：从源码运行

```bash
# 1. 克隆项目
git clone https://github.com/HazardSun/StarTrail.git
cd StarTrail

# 2. 创建虚拟环境并安装依赖
python -m venv venv
venv\Scripts\activate
pip install -r requirements.txt

# 3. 启动
python main.py
```

## 详细使用手册

### 第一次启动

1. **下载并解压** `StarTrail_v1.0.2.zip`，双击 `StarTrail.exe`。
2. **首次星历下载**：程序会自动下载 JPL 行星历表 `de421.bsp`（约 17MB）到 `data/` 目录。若下载缓慢或失败，可手动下载后放入 `data/`（见 FAQ）。
3. **设置观测城市与光污染**：进入「设置」页，选择你的城市（或自动 IP 定位），并设置光污染等级（Bortle 1–9，决定观星指数计算）。
4. **填写 OpenWeatherMap 密钥**（观星预报功能必需）：在「设置 → API 密钥」中填入，点击「测试」验证。
5. 切换至「实时星空」，开始你的第一次观星。

### 2. 观星预报

基于天气、云量、月相、湿度、大气视宁度等多维度综合评估观星适宜度，核心是一个评分仪表盘。

```
┌──────────────────────────────────────────┐
│            🌟 观星适宜度                    │
│              ╭───╮                         │
│             │ 85 │  ← 圆形仪表盘（分数+等级）│
│              ╰───╯                         │
│          ████████░░ 评分进度条             │
│                                          │
│  📊 评分规则 / 📉 今日扣分明细              │
│  🌤 当前天气详情（温度/湿度/云量/风速…）    │
│  🌧 未来降水预报（2h/8h/12h/24h）          │
│  ☀️🌙 日月升落   🌙 月相   🪐 今晚行星      │
│  🔬 专业预报参数（专业模式）               │
└──────────────────────────────────────────┘
```

**指数等级说明：**

| 等级 | 分数 | 建议 |
|------|------|------|
| ⭐ 优 | 80-100 | 非常适合观星 |
| 👍 良 | 60-79 | 适合观星，注意天气变化 |
| 👀 一般 | 30-59 | 条件一般，可尝试观测亮星 |
| ❌ 差 | 0-29 | 不适宜观星 |

**各区块说明：**
- **评分规则**：列出云量、能见度、湿度、月相、风速、光污染六项因子的扣分规则。
- **今日扣分明细**：逐项给出实测值与扣分数，直观看到「为什么扣分」。
- **当前天气详情**：温度、体感、气压、湿度、云量、能见度、风向、风速。
- **未来降水预报**：未来 2 / 8 / 12 / 24 小时是否有降水及概率，无降水显示「☀️ 无降水」。
- **日月升落**：日出 / 日落、月出 / 月落时刻（已按本地时区修正）。
- **月相 / 今晚行星**：当前月相与照明度；今晚可见行星及高度、星等。

## API 密钥配置

应用需要以下免费 API 密钥以获得完整功能：

### 1. OpenWeatherMap（观星预报必需）

- **注册地址：** https://openweathermap.org/api
- **免费套餐：** 每分钟 60 次调用
- **注意：** 该 API 密钥属于私密凭证，应用**永不**将其降级到 HTTP 明文传输（仅 ip-api、Celestrak、OpenSky 等非密钥接口会在 SSL 失败时自动回退 HTTP）。在中国大陆访问 OpenWeatherMap 可能需要科学上网，但请放心：即使开启 VPN，自动定位也不会因此覆盖你的本地坐标。

### 2. NASA API（每日一天文图）

- **注册地址：** https://api.nasa.gov
- **免费套餐：** 每小时 30 次（不配置则使用 DEMO_KEY，限额更低）

在「设置 → API 密钥」中输入密钥后点击「测试」验证连接。

## 技术栈与数据来源

### 技术栈

| 模块 | 说明 |
|------|------|
| **PySide6** | Qt6 桌面 GUI 框架 |

### 数据来源

| 数据 | 来源 |
|------|------|
| 亮星数据 | Hipparcos 星表（内置最亮恒星 + 全部命名恒星） |
| 行星位置 | JPL DE421 历表（通过 Skyfield） |
| 深空天体 | 内置目录（`data/dso_catalog.json`，含 M31/M42/M45 等） |
| 天气数据 | OpenWeatherMap API |
| 天文图片 | NASA APOD API |
| 自动定位 | ip-api.com |
| 飞机追踪 | OpenSky Network / ADS-B Exchange |
| 卫星数据 | Celestrak TLE |
| 天体资料页 | 中文维基百科 / Aladin Lite / N2YO / Flightradar24 / JetPhotos |

## 自行打包

> 本项目已随仓库提供 `build.py`，可将应用打包为单文件夹 EXE（含 `_internal` 依赖目录）。

```bash
# 安装 PyInstaller 与运行依赖
pip install -r requirements.txt

# 运行打包脚本
python build.py

# 打包后的 EXE 在 dist/StarTrail/StarTrail.exe
# 打包后的 ZIP 在 StarTrail_v1.0.2.zip
```

如需生成可分发的 ZIP，脚本会自动将 `dist/StarTrail/` 压缩为 `StarTrail_vX.X.X.zip`。

## 常见问题 FAQ

### Q: 启动后星空图是空白的？
A: 首次启动需要下载 JPL 星历表（约 17MB）。如果下载失败，请检查网络连接，或手动下载 `de421.bsp` 放入 `data/` 目录（也可从 https://naif.jpl.nasa.gov/pub/naif/generic_kernels/spk/planets/ 获取 DE421）。

### Q: 观星指数 / 降水显示「--」？
A: 请先在「设置 → API 密钥」配置 OpenWeatherMap API 密钥（观星预报与降水预报依赖此接口）。若已配置仍显示「--」，多为网络无法连通该接口所致（中国大陆可能需要科学上网）。注意：开启 VPN 不影响定位——应用会保留你的本地坐标而非 VPN 出口 IP 所在地。

### Q: 双击天体打不开网页 / 链接无效？
A: 资料页在系统默认浏览器中打开，需要联网。绝大多数资料页（维基百科、Aladin、N2YO）可直接打开；Flightradar24 / JetPhotos 对脚本访问有反爬限制，但用浏览器正常打开没有问题。

### Q: 侧边栏太窄 / 文字被截断？
A: 在左侧栏**右边缘细竖线（拖拽手柄）**上按住鼠标左右拖动即可调整宽度（180–450px）。调整后的宽度会自动保存，下次启动保持。

### Q: 如何更新？
A: 下载最新版 ZIP 解压覆盖即可。配置文件保存在 `%USERPROFILE%/.startrail/config.json`，升级不会丢失。

### Q: 专业模式卡顿？
A: 在设置中取消「自动刷新」可降低 CPU 占用。如果显卡支持 OpenGL 3.3+，可开启「GL 渲染器」获得更流畅体验（需重启生效）。

### Q: 软件会泄露我的 API 密钥吗？
A: 不会。密钥类接口（OpenWeatherMap、NASA）强制使用 HTTPS，永不降级为明文 HTTP；仅非密钥类公开接口会在 SSL 失败时自动回退，且不涉及任何凭证。
